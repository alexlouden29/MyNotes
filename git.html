Git Notes

Getting Started

  1. Create new repository on github.com. It's easier to avoid creating a new README or .gitignore at this point.
  2. Navigate to project directory in the terminal. (Use gitbash for windows)
  3. Use command "git init"
  4. Github will give you a .git URL. The next command should be "git remote add origin __youURL__.git". This links the local respository created by git init to the one on github.
  5. Use command "git add *" to stage all files for commit.
  6. Use "git commit" to commit all files to your local repository.
  7. Use "git push -u origin master" to push them to the online repo

Commmands

  • git init: Creates a new local repository in the folder from which the command is run.
  • git add: All arguments following add are staged for commit to the local repository. Edits to files that have already been added will not appear in repo if git commit is run before the file is re-added. Using "*" or "." as the argument adds all files.
    • -all or -A: If files have been deleted from the working directory, then they will be removed from the files staged for commit. Without this, a file that was previous staged and then deleted will stay staged, even if "git add ." is run.
    • --force or -f: adds all files including ones listed in the .gitignore file.
  • git commit: Save all staged files to the local repository.
    • -m: allow an argument after commit that is a comment on the commit. Ex: git commit -m "This is a commit comment"
    • -a: Commits all file updates (and deletes) whether they are staged or not. This does not however include new files not yet in the git repo. These needed to be added using git add first.
  • git status: lists files taged for commit, changes not yet staged, and the current commit that is being worked on.
  • git config --global alias.a '!__anygitcommand__' : This allows you to create your own, shorter git commands. A good use is to put two commands combined with && that you often string together, Ex: '!git add -A && git commit -m'
  • git --version: Gives the current version number of git.
  • Adding "&&" between two seperate git commands allows you to string them together on one line. Ex: git add * && git commit -m "Added and commited at same time"

A Conceptual Explanation

Work flow: Working Directory -> Staging Area(about to save) -> Repository(commited files)

Miscellaneous
