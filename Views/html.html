  • and of course "<body>""
  • example: 
    •

 	<head>

 	</head>

    Tags

    • <br> : Creates a page break
    • <blockqoute> : Creates a weird spaced and indented piece of text, like a special <p> tag.
    • <a> : Used to wrap any element and make it a clickable link.
      • href="linkURL" defines what link the user will be taken to if clicked. "#" takes the user to the top of the page, like so:
        • go to top!
      • target="_blank" opens link in a new page. Unknown yet what else can be done with target.
    • <img src=''/> : Displays the element at the location specified by src tag. Often a URL.
      • Can be used for XSS (Cross-site Scripting)
      • "alt" attribute can be used to create a description of the image, for visually impared, or if the image does not load
    • <style> : Used to put CSS elements directly into an HTML file.
    • <link> : Used to link an HTML file to another source, such as a CSS file.
      • Example: <link href="/style.css" type="text/css" rel="stylesheet"> Link elements do not have content like the <a> element. If the css file is out one folder and then in another, use ../ (like linux shell) to go back one folder.
    • <pre> : Keeps the formatting as it is typed in the html file. Used for stuff with weird formatting (and prettyprint).
    • <li>
    • <ol> and <ul>
    • <div> : A "container" for html. It can be used for visual clarity, and most powerfully, it can be used to group elements together that need common css styling. This further adds to code readability. A common class name used with divs is "container".

    Universal Attributes

    • id="idname": Use to set a unique id to an html element. This is used for selective CSS styling.
    • class="classname": Same as ID but does not need to be unique, thus groups of elements can be easily styled in the same way. Multiple class can be added by seperating them with spaces.

    The Box Model

    • The Box Model refers to the fact that every element in HTML has an unseen "box" around it that contains it all. This box can be resized with CSS.

    SVG

    • Support for event handlers

    Canvas

    • No support for even handlers

    DOM: Document Object Model

    • This is how HTML pages elements are sorted in a tree like way. Elements have parents and children. For example, a <li> element might have a <ul> element as its parent.
