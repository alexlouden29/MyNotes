Java Notes


Overview

Java focuses on security and flexibility, while being portable.

Java uses single inheritance, a class can only have one parent class. This simplifies understanding what is happening in the program.

For further simplification, Java also uses Reference instead of pointers and handles memory addresses internally. It also has a garbage collector so deallocation of memory is not an issue.

Java has a focus on security, the JVM does not allow manipulation of hard disk memory from an applet, and checks for other risky functionality.

Java is platform independent, making it useful for writing compatible software. Each platform just needs it's respective JVM, but the code does not need to change.


Java Package Terms

  • JDK: Java Development Kit, this is everything you need to write, compile, and run java code. Includes the JVM, and libraries.
  • JVM: Java Virtual Machine, this is how java runs on any platform. There is a JVM made specific to each platform, and then the JVM takes universal java and sends commands to the hardware in whatever way it needs.
  • JRE: Java Runtime Enviornment, this is what you need to run java applications. You cannot write them with this alone.
  • J2SE: This is Java 2 Standard Edition, it specifies what capabilities the package has. This one is the most standard package.
  • J2ME: Java 2 Micro Edition, used for writing code for devices that have limited space and processing power, read: phones.
  • J2EE: Java 2 Enterprise Edition. This is for the big stuff, major sites, one example being an e-commerce site. More powerful options are available.

Basic Functions and functionality

  • System.out.print.ln() : Obviously this is how you print to console, and it's the weirdest way I've seen in any language.
  • // is a single line comment, /* Is a multiline comment */
  • % is modulo, which gives the remainder of a division.
  • Relational Operators: ==, !=, <, >, <=, >=
  • Ternary conditional statement: One line if/else in java. It is actually an "expression" which means it can be put anywhere an expression can be, perhaps as the argument of a function, or embedded within a string assignment. In the example below, 'W' is executed if true, and 'L' if false. 
    • 			char gameResult = (pointsScored > 20) ? 'W' : 'L';
  • Switch: Essentially a shorthand if/else chain. Must have a default. Break is typically used after each case, without it the program will continue on, checking more cases for another match (this could have a use depending on the situation). Ex: 
    • 		switch(number){
			case 1: System.out.print("1");
				break;
			case 2: break; //Do Nothing
			default: System.out.print("Number out of range");
				 break;
		}
  • The main method: When a java program is run, THIS is what's executed. All the class definitions and what not are just tools used by the main method. Goes inside a class definition.
    • 		public static void main(String[] args){

		}
  • For loop: Repeats until condition is met. First "argument" can be used to initialize variable but can also be skipped (semicolon must still be placed). There can also be multiple initializations, comma seperated. Second is the condition, the code is run if it evaluates true. and third is code to run at the end of every loop. Thiis is typically used to increment a counter of some sort, but can be used for anything, or skipped as well. Multiple comma seperated statements are also possible. Each time after the block of code is run, the condition is re-checked.
    • 		for(int x = 0, int y = 10; x < 5 && y > 0; x++, y-=2){
			//Do something
		}
  • for each loop: Used for going through a data structure, rather than having a condition and counter it just continues until every element has been accessed one by one. 
    • 		for(int number : arrayOfNumbers){
			//number will be each element one by one
		}

Operators

  • && : The AND operator, returns a boolean true if both booleans are true, otherwise returns false.
  • || : The OR operator, returns true if either booleans are true, false if niether.
  • ! : Placed with a boolean expression, it causes the opposite boolean to be returned. Ex. !true == false.
  • Precedence: !, then &&, then || is evaluated last.
  • == : Equality, if the expressions on both sides evaluate to the same type and value, it will return true. Expressions of different data type cannot be comapred, and inequal values will return false.

Inheritance

    Classes that inherit from others are able to use their own methods as well as ones defined in their parent class (the class they inherit from). Use the keyword "extends" to indicate that a class inherits from another. Ex: 

    		class Car extends Vehicle {
			//Car class
		}

Interfaces

An interface acts as a sort of parent class that defines what functions any class that implements it will have. The main purpose is to allow a client side function to interact with any class within that type (say sorting algorithm) whithout having to change it's method calls to change which algorithm is used. It avoids specific type identification when sometimes only a more broad knowledge of the type is needed.

Example: 

		public Interface int { 
			int x = 1;
		}

Data Structures

  • ArrayList: A list of items of one similar type. If a parent class is used as the type, could have different subtypes in the same ArrayList. 
    • 	ArrayList myArray = new ArrayList();
	myArray.add(6);
	int value = myArray.get(0); //Equals 6
	myArray.add(0, 7)  //Array = [7,6]
  • ArrayList Functions
    • .add(index(opt), value): When an index is given for add, all present values shift up to make room for the new value. Otherwise, it adds to the end of the list. Eg. add(4), or add(2,4)
    • .get(index): Returns the value at index in the array.
    • .size(): Returns the size of the array as int.

  • HashMap: A bunch of key, value pairs. Keys can be of any one type, and value pairs of any one type, they do not need to be the same. Essentially if you have the key, you can get the value.
    • 		HashMap myHashMap = new HashMap();
  • HashMap Functions
    • .put(key, value): does what it says, adds a key value pair to a hash map.
    • .get(key): Returns the value associated with the provided key.
    • .size(): returns the number of pairs in the table (DUH).
