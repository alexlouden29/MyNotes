CSS Notes

Cascading Style Sheets

  • CSS can go into a style tag, but better in a .css file for organization
    • To put it in a seperate file, must include a "link" HTML element to connect them

    Breaking Down the syntax 

     
		h1 {
  			color: Blue;
		}
    • The whole bracketed thing is called a rule
    • The first part is the selector, it chooses which elements to change. Can have multiple selectors for one rule by seperating them with commas. Applies to all types of selectors.
      • "*" is the universal selector
      • .button can be used to select any element with class="button". The . selects by class name
      • #green <-- # sign is used to select by id, where id = "green"
      • List of all selectors There are a ton of them. Interesting to me is the ":visited" selector, which affects all visited links.
      • standard and class selectors can be combined. The following examples sets the font family of all elements of class breaking, and then changes the line height of paragraph elements that are of class breaking. 
        • 	.breaking {
	  font-family: Georgia, Times, serif;
	}

	p.breaking {
	  line-height: 1.3em;
	}
    • An individual line ended by a semicolon is a "declaration" e.g. color: blue;
    • A Declaration is made up of a property and a value, color is the property, blue is the value.
    • /* Comment */ is how you comment.
    • If there are multiple declarations, the latter one takes affect. This can be used to improve browser support. For example: 
      • 		h1 {
  			color: rgb(22, 34, 88);
  			color: rgba(22, 34, 88, 0.4);
		}

      In the above example, a newer browser will overwrite the first rgb with the rgba, while older ones just ignore rgba and stick with the rgb.

    • On the topic of css precedence: inline (style="a style") css takes first precedence. After that, the more specific the rule, the higher the weight. If the specificity is the same, rules later (further down) in the css file, or lower on the html style tag, overwrite the earlier ones.

    Possible Properties and Values

    • color: change the color of the element selected
      • Blue
      • Green
      • Red
      • FireBrick
      • DarkSlateGray
      • MidnightBlue
      • The rest of them
      • use rbg(red value, blue value, green value) with all parameters as ints 0-255 to create any color. Fun fact, there are 16,777,216 options. Go to this color picker to figure out the values.
      • rbga(red, blue, green, opacity): opacity is a value from 0 to 1.
      • Hex values can also be used, a la color: #FFFFFF, with pairs representing colors. This can be shortened like so: #FF AA FF -> #F a F -> #FAF = #FFAAFF
      • or, hsl(hue, saturation, lightness) : 0-360 for hue, and a percentage for the other two. Saturation is percent gray, lightness is percent white. Only supported on newer browsers. Ex. hsl(12, 10%, 50%) (Why would anyone use this?)
      • hsla(): same as rbga
      • rgba, hsl, hsla not supported in old browsers.
    • background-color: change the background color of the html element. Same color options as above.
    • font-size: Changes the size of the text
      • By pixel: 16px
      • By ems: 1.3em em is a relative size based on the default size of the letter m.
        • A note on ems. They seem to take precendence these days, because of their strength with relative sizing. If you use ems, you only need to change the font size of the body element if you change screensize (ie. mobile vs. desktop) and everything with relatively cascade down or up after that change. px kinda does this too, but this works with margins as well. To avoid cascading, eg. with nested lists, you can do li li {font-size: 1em } to normalize the nested lists, so that the parent effect is the total effect.
      • By percentage: 125% Percentage scales off of the default size, 16px.
    • line-height: Change the spacing between hights. Can use px or em, em is preferred. 
      • 	p {
		line-height: 1.5em;
	}
    • Fun fact: Line height is the font size plus the "leading" (ledding) which is the spacing between the lines.
    • word-spacing: Space between words! Again px or em, em is preferred. On the whole, this is uncommon.
      • 	h1 {
  		word-spacing: 0.3em;
	}
    • letter-spacing: Just take a guess. Same details as word spacing, technical term is "kerning" for god knows what reason. 
      • 	h1 {
  		letter-spacing: 0.3em;
	}
    • font-weight: Can change to bold, OR set everything to bold, and use "normal" to "switch off bold".
      • bold
      • normal
      • 100(thin), 200, 300, 400 (normal), 500, 600, 700, 800, 900(bold) Yes, just those exact numbers, and I don't know why.
    • font-style: Guess based on the options below
      • italic
      • normal (for same reason as font-weight normal option)
    • text-transform: Main purpose being uniformity, or avoid typing actual uppercase in the HTML file.
      • uppercase
      • lowercase
    • text-align: default is left align.
      • left
      • center
      • right
    • font-family: Changes the font
      • For two word fonts, Quotation marks must be used. Eg. "Times New Roman"
      • Default typeface(font) for all HTML: Times New Roman.
      • Fun fact, Serif fonts have the little details on the ends, while Sans-Serif fonts do not.
      • Fallback font: To use a font listed in a CSS file, a computer must have that font installed. Fallback fonts can give 2nd and 3rd string options to computers who are short on fonts. 
        • 		
		h1 {
  			font-family: Garamond, Times, serif;
		}

        serif simply means, any serif font installed on the computer.

      • Google Fonts is a collection of open source fonts, and can be used by any computer if a link to that specific font is in the HTML. Other font collections exist. Link like so:
        • 				<link href="https://fonts.googleapis.com/css?family=Roboto:100" type="text/css" rel="stylesheet">
    • width: Changes the width of the box around an element.
      • Can use px, ems, or percentage. For percentage, it's a percent of the size of the parent box (the box that this box is inside, the largest being the body.)
      • height: The height of the box around an element, with the same value options as width.
      • min-width, max-width, min-height, max-height: In px, sets the minimum and maximum width of an element, to avoid strange resizing issues, and keep text readable. If the max-height is too low, contents might spill out. A lower max-width could contribute to this problem.
      • overflow: Determines what happens if content spills out of its box because of max width and height settings.
        • scroll: a scroll bar appear and the spilled content can be viewed by scrolling.
        • hidden: spilled content will simply not be visible.
